# 三大核心难点解决方案

---

## 难点一：角色一致性

跨镜头/跨场景中角色外观漂移，是AI漫剧的头号杀手。

### 解决方案：三层锚定架构

**第一层：输入端锚定（预防）**

- 参考表系统（Reference Sheet）：每个角色生成3-5张中性姿态参考图（正面/侧面/3/4角/背面），纯白背景，统一服装发型
- 结构化角色描述卡：不是自由文本，而是固定字段模板
  ```
  角色ID: sanguan_01
  姓名: 商三官
  性别: 女（男装时）
  面部: 鹅蛋脸，细长眉，眼尾微翘，鼻梁挺直，薄唇
  发型: 女装-长发高髻银簪 / 男装-束冠黑巾
  服装: 女装-月白襦裙淡青披帛 / 男装-靛蓝圆领袍黑色革带
  体型: 纤瘦，165cm
  标志特征: 左眉尾一颗小痣
  ```
- 每次文生图/图生视频提示词中**强制完整重复角色描述卡关键字段**

**第二层：生成过程约束（控制）**

当前可用的技术手段，按一致性从强到弱：

| 手段 | 一致性 | 成本 | 适用场景 |
|------|--------|------|---------|
| LoRA微调 | 最高（95%+） | 高（需训练） | 主角，出场>20集 |
| 参考图绑定（IP-Adapter） | 高（85-90%） | 中 | 所有主要角色 |
| @提及系统（Seedance 2.0） | 高 | 低 | 即梦生态内 |
| 提示词完整描述重复 | 中（60-75%） | 极低 | 所有场景兜底 |
| ControlNet姿势引导 | 中 | 中 | 特定动作场景 |

推荐组合策略：
- **主角**：LoRA微调 + 提示词重复（双保险）
- **主要配角**：参考图绑定（IP-Adapter / Seedance @图片） + 提示词重复
- **次要角色**：提示词重复即可

**第三层：输出后校验（修复）**

- art_qa Agent 对每张生成图做跨图比对，检测：
  - 面部特征漂移（五官比例/脸型变化）
  - 服装不一致（颜色/款式突变）
  - 发型变化
- 不合格图标记为 reject，附带修正提示词退回 video_producer 重生成
- 严重漂移时，使用局部重绘（inpainting）修复而非全图重生成

### 工具链推荐

| 环节 | 推荐工具 | 备注 |
|------|---------|------|
| 参考图生成 | Midjourney V7 / Flux Kontext | 一致性最强 |
| 文生图（带参考图绑定） | 即梦3.0 Pro / Seedance 2.0 | @图片绑定角色 |
| 图生视频（带参考图绑定） | Seedance 2.0 / 可灵3.0 | 角色ID锁定 |
| LoRA训练 | Kohya_ss / 即梦LoRA | 主角专用 |
| 一致性校验 | art_qa Agent（视觉比对） | 自动化质检 |

## 难点二：提示词质量

提示词精准度直接决定：1）生成质量 2）抽卡次数 3）成本

### 解决方案：结构化提示词模板 + 质量门控

**文生图提示词模板（7段式）**

```
{景别}，{角色完整描述}，{主体动作/状态}，{环境场景}，{情绪氛围}，{视觉风格}，{光影效果}
```

示例：
```
中景，商三官（鹅蛋脸细长眉眼尾微翘左眉尾小痣靛蓝圆领袍束冠黑巾纤瘦），站立抚琴，
赵府偏厅红木琴台烛光摇曳，沉静中暗藏紧张，古风写实工笔画风，暖黄烛光侧光阴影分明
```

**图生视频提示词模板（5段式）**

```
{镜头描述}，{镜头运动}，{主体行为}，{背景变化}，{环境变化}
```

示例：
```
中景固定镜头，缓慢推进，商三官手指拨动琴弦微微低头，烛火随气流轻晃，窗外夜色深沉
```

**提示词铁律**：

1. 文生图描述静态瞬间，图生视频描述动态——不能混用
2. 一镜一事，不要在一个镜头里塞多个动作
3. 先静后动，少量慢移动——AI视频最怕快速复杂动作
4. 适度语义模糊——过细描述（"左手食指轻触第三根琴弦"）会触发AI幻觉
5. 必须包含负面提示词：毁容、面部扭曲、肢体变形、多余手指、模糊
6. 每次必须完整重复角色外观描述——不能写"同上"或省略

**质量门控：storyboard_qa Agent**

分镜脚本提交后，storyboard_qa 逐镜头检查：

- [ ] 是否包含完整景别
- [ ] 角色描述是否完整重复（不能省略）
- [ ] 文生图提示词是否描述静态（不含动态动词）
- [ ] 图生视频提示词是否描述动态
- [ ] 一镜是否只做一件事
- [ ] 时长是否≤10秒
- [ ] 是否包含负面提示词
- [ ] 动作链是否逻辑连贯

不通过的镜头打回重写，不进生产环节。

## 难点三：镜头级生产精度

不能整集一键生成，必须逐镜头生产再合成。80集×20镜头/集=1600个镜头，每个都需要独立的生产-质检-修复循环。

### 解决方案：流水线化 + 重试预算制

**流水线架构**

```
                    ┌─── storyboard_qa ──┐
storyboard ────────►│                    ├─── 通过 ──► video_producer
                    └─── 打回重写 ◄──────┘
                                            │
                                    ┌─── video_qa ──┐
                                    │               ├─── 通过 ──► 下一镜头
                                    └─── 打回重试 ◄─┘
```

- 同一集内，镜头按顺序生产（前后镜头有连贯性依赖）
- 不同集之间，可流水线并行（ep01视频制作时，ep02分镜已开始）

**重试预算制**

每个镜头设定最大重试次数，避免无止境抽卡：

| 镜头类型 | 最大重试 | 原因 |
|---------|---------|------|
| 静态文生图 | 5次 | 成本低，一致性可控 |
| 简单动态视频（走/坐/看） | 5次 | 动作简单，成片率高 |
| 复杂动态视频（打斗/舞蹈） | 8次 | 成片率低，需要更多尝试 |
| 对口型镜头 | 3次 | 口型匹配难，超限则降级为画外音处理 |

超过预算仍未通过的镜头：
1. 标记为 problem_shot
2. 上报 director Agent 决策
3. director 可选：简化动作重写 / 降级处理（静态图+旁白替代） / 手动干预

**成本估算（单集）**

```
20镜头/集
× 5次平均重试
× 文生图：~0.5元/次（即梦3.0）
× 图生视频：~2元/次（即梦3.0 Pro 10秒）
= 文生图：50元/集
= 图生视频：200元/集
= 单集视频制作总成本：~250元
= 80集全剧：~20000元
```

**分段生产策略**

不要一次生产整集，按"镜头组"分段：

```
镜头1-5（场景A）→ 质检 → 修复 → 通过
镜头6-10（场景B）→ 质检 → 修复 → 通过
镜头11-15（场景A）→ 质检 → 修复 → 通过
镜头16-20（场景C）→ 质检 → 修复 → 通过
```

同场景的镜头组一起生产，共用参考图和场景设定，一致性更高。相邻镜头组切换场景，避免空间布局冲突。

## 三个难点的交叉影响

```
提示词质量差 → 抽卡次数多 → 成本飙升
     ↓               ↓
角色不一致 → 视频质检不通过 → 返工 → 成本再飙升
     ↓
镜头级精度不够 → 成片质量差 → 导演打回 → 全流程返工
```

**核心原则：上游严格 = 下游省钱**

- 剧本阶段写好人物圣经 → 美术阶段就能生成高质量资产 → 分镜提示词就有好基础 → 视频生成就少抽卡
- 任何环节偷懒，下游都要加倍还债
