# 各环节模型选型（2026-06-21 调研）

> 针对本项目流水线的每个环节，给出当前最新、最合适的模型推荐。
> **数据来源**：2026-06-21 web 调研；排名/口碑多为第三方评测、偏主观，**未经本项目实测**。最终取舍建议在 ep01 上各跑一遍再定。
> 云端模型为质量上限，本地为免费/私密替代。OpenAI 兼容的可直接改 `config.yaml`；其余需在对应 executor 写 `_call_*`。

## 创意层（文本 LLM）

| 环节 | 首选（云） | 理由 | 本地/备选 | 接入 |
|---|---|---|---|---|
| writer 编剧 | **Kimi K2.6**（月之暗面） | 中文创意写作口碑最佳、长文稳 | DeepSeek-V4（便宜，文笔稍硬）/ 本地 Gemma | OpenAI 兼容，改 config |
| storyboard 分镜 | **DeepSeek-V4** 或 **GLM-5.2** | 结构化/指令遵循稳，吐 `###镜头` 格式可靠 | 本地 Gemma / Qwen3.6 | OpenAI 兼容 |
| director 终审/升级 | **DeepSeek-V4** / **Qwen3-Max-Thinking** | 判断/推理强 | 本地 Gemma | OpenAI 兼容 |
| discovery 选题评估 | **DeepSeek-V4** / **GLM-5.2** | 长文阅读 + 评估推理 | 本地 Gemma | OpenAI 兼容 |

> 省钱混搭：writer 用 Kimi（质量敏感），其余用本地 Gemma 或便宜的 DeepSeek。

## 质检层（视觉 LLM）

|---|---|---|
| visual_qa 画面质检 | **Qwen3-VL / Qwen3.5-Plus**（阿里，最强视觉理解）或 **GLM-4.5V**（智谱） | 本地 Qwen-VL / Gemma 3 多模态 |

## 执行层（生成模型）

| 环节 | 首选（云） | 理由 | 本地/备选 |
|---|---|---|---|
| text2img 文生图 | **即梦 Seedream** / **可灵图片3.0 Omni** | 支持参考图、角色一致性好、2K/4K | **Flux + LoRA/IP-Adapter**（本地免费，一致性生态最成熟） |
| img2video 图生视频 | **Seedance 2.0**（字节，多参考图最多7主体、适合短剧/动漫、均衡可控）；或 **可灵3.0**（表现力强、登顶榜） | 短剧最看重跨镜头一致性 | 本地 **万相/Wan**（吃显卡） |
| audio 配音 | **MiniMax Audio（speech-2.8-hd）**（情感/多角色表现力最强） | 短剧多角色情感台词 | 本地免费 **CosyVoice**（阿里开源）；现状 edge-tts（够用基线） |

## 角色一致性（贯穿 text2img/img2video，本项目头号难点）
- 云端：选 text2img/img2video 时**优先选支持「参考图/多主体绑定」的**：即梦 Seedream、可灵 Omni、Seedance（多参考图）。

### LoRA 一致性（本地路线，2026-06 调研）
- **首选 Qwen-Image（阿里，开源）+ Qwen-Image-Edit**：
  - 中文/古风原生，比 Flux（西方训练）更对路本项目题材；
  - `Qwen-Image-Edit` 能**从一张定妆图生成同角色多角度数据集**，自动化"3-5 张多角度参考图"这一步，再拿去训 LoRA；
  - 开源、本地、免费。
- **强力备选 FLUX.2（Black Forest Labs，如 klein 9B）**：画质 + LoRA 生态最成熟（"企业级一致性"），但古风/中文不一定有 Qwen-Image 地道，且 9B 吃显卡。
- **工具链**：训练用 Kohya_ss / ai-toolkit；编排用 ComfyUI（角色一致性工作流事实标准）。
- **流程**：主角定妆图 → Qwen-Image-Edit 出多角度参考集 → 训角色 LoRA → 每镜头带 LoRA 出图 → 跨镜头稳定。接进本系统即 `text2img` 的本地 provider。
- **策略**：主角（商三官）单独训 LoRA；配角用 IP-Adapter/参考图即可，不必每个都训。
- 注：以上为 2026-06 社区/评测信号，未在本项目实测；古风效果与显存占用需真跑验证（建议 Qwen-Image vs FLUX.2 各训一个小 LoRA，同角色 5 镜头比一致性）。

## 两条产品路线对应的最小模型组合
- **漫剧（静态画面+运镜+配音，最快出真片）**：Kimi/DeepSeek（创意）+ 即梦 Seedream 或本地 Flux（文生图）+ MiniMax 或 CosyVoice（配音）+ ffmpeg。**不需要图生视频模型。**
- **真·短剧（画面动起来）**：在上面加 **Seedance 2.0 / 可灵3.0**（图生视频），成本和难度上一台阶。

## 接入方式速记
- **OpenAI 兼容（改 config 即可）**：Kimi、DeepSeek、Qwen、GLM、各家文本/视觉 LLM。注意 `api_key` 必须非空，否则触发本系统离线模式。
- **各家私有 API（需写 executor `_call_*`）**：即梦/Seedream、可灵/Seedance、MiniMax TTS。
- **本地（OpenAI 兼容服务或自带推理）**：Gemma/Qwen（Ollama 等）、Flux/SD（ComfyUI）、CosyVoice。

## 待办（落地选型）
1. 先接**本地 Gemma**（创意层）跑通真实内容 → 不满意再单独给 writer 挂 Kimi/DeepSeek
2. 文生图先上**本地 Flux** 或**即梦 Seedream** → 出真实漫剧画面
3. 配音从 edge-tts 升级到 **MiniMax / CosyVoice**
4. 要真视频再接 **Seedance 2.0 / 可灵3.0**
