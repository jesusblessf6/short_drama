# AI漫剧全Agent自动化架构

## 一、8岗角色模型

| 岗位 | Agent名 | 核心职能 | 上游输入 | 下游输出 |
|------|---------|---------|---------|---------|
| 制片人 | producer | 选题决策、预算、进度、合规、投流；终审（与导演联合） | 用户需求、市场数据 | 项目立项书、进度表 |
| 导演 | director | 全局创意把控、节奏情绪、每集成片终审；终审（与制片联合） | 剧本+分镜+成片 | 创意方向、审核意见、修改指令 |
| 编剧 | writer | 工程化剧本（总纲+人物圣经+场景圣经+分集剧本） | 框架+原著 | 分集剧本 |
| 美术设定 | art_director | 角色/场景/道具资产包构建，风格统一 | 人物圣经+场景圣经 | 资产包（参考图+风格定义） |
| 分镜设计 | storyboard | 拆分镜+文生图/图生视频提示词编写 | 剧本+资产包 | 分镜脚本（含提示词） |
| 视频制作 | video_producer | 文生图执行+图生视频+抽卡筛选 | 分镜脚本+资产包 | 视频片段 |
| 音效 | audio | AI配音+对口型+BGM+音效 | 剧本（台词）+视频片段 | 音轨（配音+BGM+音效） |
| 剪辑 | editor | 拼接+字幕+转场+调色+成片输出 | 视频片段+音轨 | 成片 |

## 二、分布式质检架构

质检不单独设岗，每个环节都有对应质检：

```
┌─────────────────────────────────────────────────────┐
│                    质检流水线                         │
├─────────────┬──────────────────┬────────────────────┤
│   生产环节   │    质检方式       │    质检重点         │
├─────────────┼──────────────────┼────────────────────┤
│ 编剧→剧本    │ writer_qa Agent  │ 节奏/钩子/人物一致性 │
│ 美术→资产    │ art_qa Agent     │ 跨图一致性/风格统一  │
│ 分镜→脚本    │ storyboard_qa    │ 提示词完整性/镜头逻辑│
│ 视频→片段    │ video_qa Agent   │ 穿帮/变形/漂移      │
│ 音效→音轨    │ audio_qa Agent   │ 音画同步/自然度      │
│ 剪辑→成片    │ editor自检       │ 节奏/转场/时长       │
│ 每集成片     │ 导演终审          │ 整体创意/情绪/节奏   │
│ 全剧成片     │ 导演+制片联合终审  │ 整体完成度/合规/商业  │
└─────────────┴──────────────────┴────────────────────┘
```

**原则：上游质检严格度 > 下游**。剧本返工成本极低，视频返工成本极高。

## 三、Agent编排流程

### 3.1 宏观流程（全剧级别）

```
Phase 1: 立项
  producer ─→ 确定选题、预算、排期 ─→ 项目立项书

Phase 2: 前期创作
  writer ─→ 工程化剧本 ─→ writer_qa ─→ 修改/通过
  art_director ─→ 资产包 ─→ art_qa ─→ 修改/通过

Phase 3: 中期生产（逐集流水线）
  storyboard ─→ 分镜脚本 ─→ storyboard_qa ─→ 修改/通过
  video_producer ─→ 视频片段 ─→ video_qa ─→ 修改/通过
  audio ─→ 音轨 ─→ audio_qa ─→ 修改/通过
  editor ─→ 成片 ─→ director终审 ─→ 修改/通过

Phase 4: 终审与发行
  director + producer ─→ 全剧终审 ─→ 合规/投流
```

### 3.2 微观流程（单集生产）

```
writer(ep脚本) ─→ writer_qa ─→ 通过
      ↓
art_director(本集所需资产确认) ─→ art_qa ─→ 通过
      ↓
storyboard(拆分镜+写提示词) ─→ storyboard_qa ─→ 通过
      ↓
video_producer(文生图→图生视频→抽卡筛选) ─→ video_qa ─→ 通过
      ↓                                       ↓ 不通过
audio(配音+BGM+音效) ─→ audio_qa ─→ 通过    返工到对应环节
      ↓
editor(剪辑合成) ─→ 自检
      ↓
director(每集终审) ─→ 通过 / 修改指令退回对应环节
```

### 3.3 并行策略

以下环节可并行：
- **美术资产**与**前N集剧本**可并行（资产包只需人物圣经即可开始）
- **视频制作**与**音效**可并行（音效只需剧本台词，不依赖视频片段）
- 不同集的生产可流水线化：第N集视频制作时，第N+1集分镜已开始

```
时间 →
ep01: [分镜] → [视频] → [音效] → [剪辑] → [导演审]
ep02:        [分镜] → [视频] → [音效] → [剪辑] → [导演审]
ep03:               [分镜] → [视频] → ...
```

## 四、Agent Persona定义

### 4.1 producer（制片人）

```
角色：制片人
职责：
  - 选题决策：评估IP短剧适配度（冲突/反转/爽感闭环）
  - 预算控制：跟踪每环节成本，视频生成是大头
  - 进度管理：推动各环节交付，处理卡点
  - 合规前置：备案、AI商用权、片头标注
  - 终审：与导演联合审核全剧成片
  - 发行：投流策略、封面标题、数据复盘
输出物：项目立项书、进度表、合规清单、投流方案
```

### 4.2 director（导演）

```
角色：导演
职责：
  - 全局创意方向：确定视觉风格、叙事调性、情绪节奏
  - 审核把关：每集成片终审，给出修改指令
  - 终审：与制片联合审核全剧成片
  - 关键决策：结局处理、名场面调度、全剧情绪弧线
输出物：创意方向文档、审核意见、修改指令
```

### 4.3 writer（编剧）

```
角色：编剧
职责：
  - 工程化剧本输出：
    1. 项目总纲（核心冲突+幕结构+节奏策略）
    2. 人物圣经（外貌/性格/服装/发型/说话方式/情绪谱系）
    3. 场景圣经（各场景的视觉/氛围/光影描述）
    4. 镜头风格说明
    5. 分集剧本（场景描述+台词+情感节点+钩子）
  - 遵循短剧节奏铁律：
    前3秒定生死 / 集尾必有钩子 / 台词极简 / 无废戏
输出物：工程化剧本全套
质检：writer_qa（节奏/钩子/人物一致性/改编内核保留）
```

### 4.4 art_director（美术设定）

```
角色：美术设定
职责：
  - 基于人物圣经，生成角色资产包：
    3-5张多角度参考图（正面/侧面/3/4角度）
    统一服装、发型、面部特征
  - 基于场景圣经，生成场景资产包：
    概念图 + 光影风格定义
  - 道具资产：纳入标准库，统一质感/年代感
  - 全局风格定调：调色方案、画风统一
输出物：角色/场景/道具资产包、风格定义文档
质检：art_qa（跨图角色一致性/风格统一/与人物圣经匹配度）
```

### 4.5 storyboard（分镜设计）

```
角色：分镜设计
职责：
  - 将分集剧本拆解为15-25个镜头/集
  - 每个镜头输出：
    1. 景别（远景/中景/近景/特写）
    2. 角色描述+动作链
    3. 文生图提示词
    4. 图生视频提示词
  - 提示词规则：
    · 必须完整重复主角外观描述（保一致性）
    · 文生图描述静态瞬间，图生视频描述动态
    · 一镜一事，先静后动
    · 适度语义模糊，避免AI幻觉
输出物：分镜脚本表（镜头号/时长/景别/文生图提示词/图生视频提示词/备注）
质检：storyboard_qa（提示词完整性/镜头逻辑/动作链连贯/时长合规）
```

### 4.6 video_producer（视频制作）

```
角色：视频制作
职责：
  - 执行文生图：根据提示词+资产参考图批量生成分镜图
  - 执行图生视频：分镜图作首帧，生成5-10秒视频片段
  - 抽卡筛选：多生成选最优，不合格重试
  - 成本控制：控制每镜头生成次数，记录消耗
  - 视频提示词调优：根据生成结果微调提示词
输出物：分镜图集、视频片段集
质检：video_qa（角色一致性/穿帮/肢体变形/漂移/物理合理性/光影稳定）
```

### 4.7 audio（音效）

```
角色：音效
职责：
  - AI配音：音色选择/克隆、情感调节
  - 对口型：配音与视频口型对齐
  - BGM：选曲或AI生成，情绪匹配
  - 音效：环境音/动作音/转场音
  - 版权合规：使用无版权音乐库
输出物：配音音频、BGM、音效、完整混音音轨
质检：audio_qa（音画同步/配音自然度/情感匹配/音量平衡）
```

### 4.8 editor（剪辑）

```
角色：剪辑
职责：
  - 视频片段拼接+转场
  - 音画合成
  - 字幕添加
  - 统一调色
  - 片头片尾
  - 单集时长控制（1-3分钟）
  - 自检后提交导演终审
输出物：成片
质检：自检（节奏/转场/时长/字幕）→ 导演终审
```

## 六、质检隔离原则

**核心铁律：质检Agent与生产Agent必须完全隔绝。**

### 6.1 三重隔离

1. **上下文隔离**——质检Agent看不到生产Agent的思考过程，只看最终输出物
2. **出发点隔离**——生产Agent是"创作者心态"（想方设法做好），质检Agent是"挑刺者心态"（想方设法找问题）
3. **知识隔离**——质检Agent不加载生产Agent的persona、技巧、经验；反之亦然

### 6.2 具体实现

- 质检Agent用独立的delegate_task子代理运行，不复用生产Agent的session
- 质检Agent的system prompt从零构建，站在"严苛审查者"立场
- 质检Agent读取的输入仅为：输出物文件 + 行业标准/规范文档
- 质检Agent不读取生产Agent的任何中间过程、草稿、思路说明
- 生产Agent不读取质检Agent的内部推理，只读取质检报告的最终结论

### 6.3 各质检Agent的独立立场

| 质检Agent | 立场 | 审查视角 |
|-----------|------|---------|
| writer_qa | 严苛剧本医生 | "这个剧本放在抖音上3秒会不会被划走？" |
| art_qa | 视觉一致性审计员 | "这两张图像不像同一个人？" |
| storyboard_qa | 镜头逻辑检察官 | "这个提示词AI能执行吗？动作链有断裂吗？" |
| video_qa | 画面穿帮猎人 | "手是不是多了一根手指？人物是不是漂移了？" |
| audio_qa | 音画同步校准师 | "嘴型和声音对得上吗？情感对吗？" |

### 6.4 反模式（绝对禁止）

- 质检Agent说"作者意图是xxx所以这里可以接受"——不允许替生产Agent解释
- 质检Agent和生产Agent使用相同的评估框架——不允许"同构审查"
- 质检Agent看到生产Agent的草稿/中间版本——不允许"过程污染"

## 七、Agent间通信协议

Agent之间通过文件系统通信（符合用户偏好的文件分片架构）：

1. **每个Agent只写自己目录下的文件**
2. **只读上游目录**
3. **质检Agent写质检报告到 08_质检/ 下对应子目录**
4. **修改指令通过质检报告中的 `action: rewrite/patch` 标记传达**
5. **状态标记**：每个输出文件头部有 `status: draft/reviewed/approved` 字段

## 八、自动化核心难点与应对

| 难点 | 应对策略 |
|------|---------|
| 角色一致性 | 资产包参考图绑定 + 每次提示词完整重复角色描述 + art_qa跨图比对 |
| 提示词质量 | storyboard_qa校验完整性 + video_producer反馈调优 + 负面提示词模板 |
| 视频返工成本 | 上游质检严格（剧本>分镜>视频），分镜脚本必须通过才进视频制作 |
| 抽卡成本控制 | 每镜头设定最大重试次数，超过则升级给director决策 |
| 音画同步 | audio_qa专项检查，口型不匹配退回audio重做 |
| 全局风格统一 | art_director输出风格定义文档，所有Agent遵守 |
